Harness Layer — Prompt + Playbook Learning

Learns from failed episodes · Evolves system prompts · Accumulates reusable strategies

1Ingest

Failed Episodes Arrive

Support-query split: only episodes with effective_reward() < 0 reach the Harness. Successful episodes go to Weights. Episodes stamped with scored_at_generation — stale episodes (older generation) are skipped.

↓
2Reflect

Reflector → Insights

LLM analyzes episode traces + current playbook. Extracts general strategies (not task-specific answers). Outputs Insight objects with action (add/update/remove), content, tags, source episode IDs.

Max 3 insights per batch · 5 episodes per prompt · Injection pattern detection · Content length cap (2000 chars)
↓
3Curate

Curator — Retrieve · Classify · Revise

Retrieve: embed new insight, find similar existing entries by cosine similarity.
Classify: heuristic-first — sim >0.95 = identical (skip), >0.8 + contradictions = conflict, 0.6–0.8 = LLM classifies, <0.6 = unrelated (add).
Revise: conflicts → create resolved entry, mark originals superseded_by. Complementary → merge. Identical → boost existing score.

Fallback: if embeddings/LLM unavailable → direct add with warning · Never hard-fail
↓
4Evolve

GEPA Pareto Front — Prompt Evolution

Mutation: LLM reads failing episodes + current best prompt → proposes targeted fix → new candidate with parent_id and generation+1.
Crossover: combines strengths of two non-dominated candidates.
Selection: Pareto front prunes dominated candidates. Best (highest mean score) is promoted to active system prompt.

↓
5Apply

optim_step — Atomic Application

Snapshot playbook + prompts + Pareto fronts (deepcopy). Apply all pending signals and insights. Auto-prune entries with negative effective_score(). Enforce max entry cap. On any failure → rollback to snapshot.

effective_score = (helpful − harmful) × exp(−decay_rate × age_days) · Entries decay from last_activated
↓
6Render

Playbook → System Prompt Injection

Active entries rendered as structured Markdown (name, when-to-trigger, content, anti-patterns) and appended to the base system prompt. Superseded entries hidden. The agent sees the evolved prompt + accumulated strategies on every future call.

Background processes: Consolidation — periodic clustering + merging of related entries, contradiction resolution, budget enforcement. Paradigm Breakthrough — when stagnation detected (reward variance < 0.02 over 5 iterations), LLM proposes fundamentally new strategic directions. Old entries marked superseded.
