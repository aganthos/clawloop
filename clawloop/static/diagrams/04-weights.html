Weights Layer — RL Fine-Tuning

Learns from successful episodes · Trains via GRPO + custom algorithms · Gate before deploy

1Collect

Successful Episodes → Training Buffer

Support-query split: only episodes with effective_reward() ≥ 0 reach the Weights layer. Each episode carries per-token logprobs (log π_θ(a|s)) for importance weighting. Episodes stamped with scored_at_generation — when Harness playbook evolves (generation advance), all stale episodes are flushed from the buffer.

↓
2Train

forward_backward → Compute Advantages

GRPO: group-relative advantage estimation — compare within sample groups, no value function needed. Custom algorithms supported via paradigm config selector. Training delegated to backend (SkyRL, Harbor, Tinker, or MinT). LoRA adapters keep base model frozen.

↓
3Gate

gate_for_deploy — Regression Check

Before promoting new weights: evaluate on held-out tasks, compare against current deployed weights. Distributional comparison ensures no regression on any task category. Only promote if the new weights are non-worse across all dimensions.

↓
4Deploy

optim_step → Hot-Swap Weights

Atomic application: snapshot current state, apply new LoRA adapter, verify. On any failure → rollback to snapshot. Content-addressed StateID ensures reproducibility — every configuration is uniquely identified by its hash.

↓
5Monitor

Continuous Evaluation

Post-deploy: continue collecting episodes with new weights. If performance degrades, AdaptiveIntensity detects stagnation and triggers more aggressive learning. If catastrophic: rollback to last known-good snapshot.
